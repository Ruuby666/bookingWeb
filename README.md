## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/Ruuby666/bookingWeb.git
cd bookingWeb
```

### 2. Configure your environment file

```bash
cp .env.example .env
```

Open `.env` and fill in the required values (see [Environment Variables](#environment-variables)).

### 3. Run the setup script

```bash
./setup.sh
```

This single command will:

1. Tear down any existing containers and volumes (`docker compose down -v`)
2. Start all Docker containers
3. Wait for MySQL to be ready
4. Install Composer dependencies
5. Generate the application key
6. Run all database migrations
7. Seed the admin user
8. Create the `public/storage` symlink
9. Set correct file permissions
10. Download property images from GitHub Releases into the Docker volume

Once complete, visit:

```
http://localhost:8000
```

Default admin credentials (defined in your `.env`):

```
Email:    ADMIN_EMAIL value
Password: ADMIN_PASSWORD value
```

> **Note:** The setup script requires Git Bash or WSL on Windows. It does not work with plain Command Prompt or PowerShell.

Copy `.env.example` to `.env` and fill in the required values:

## Database

Property images are **not stored in the repository**. They live in a Docker named volume (`images_data`) that is shared between the `app` and `web` containers and is automatically populated by `setup.sh` from a GitHub Release asset.

### How it works

```
GitHub Releases (images_v1/imagens.zip)
        в†“  curl inside container
storage/app/public/images/          в†ђ Docker volume: images_data
        в†“  php artisan storage:link
public/storage/                     в†ђ symlink served by Nginx
```

### Adding new images

Upload images from the admin panel at `/properties/{id}/edit`. They are stored directly into the Docker volume.

### Updating the release asset

When property images change significantly, create a new GitHub Release and update the URL in `setup.sh`:

```bash
https://github.com/Ruuby666/bookingWeb/releases/download/images_v2/imagens.zip
```

## Deployment

The project is deployed on a **VPS with Docker Compose** and Nginx as a reverse proxy.

### Server requirements

- Docker + Docker Compose
- Domain pointing to the server IP
- SSL certificate (recommended: [Caddy](https://caddyserver.com/) or Certbot)

### Steps

```bash
git clone https://github.com/Ruuby666/bookingWeb.git
cd bookingWeb

cp .env.example .env
# Edit .env: set APP_ENV=production, APP_DEBUG=false, APP_URL, DB credentials, MAIL, etc.

docker compose up -d --build
docker compose exec app composer install --no-dev --optimize-autoloader
docker compose exec app php artisan key:generate --force
docker compose exec app php artisan migrate --force
docker compose exec app php artisan db:seed --force
docker compose exec app php artisan storage:link
docker compose exec app php artisan config:cache
docker compose exec app php artisan route:cache
docker compose exec app php artisan view:cache
docker compose run --rm vite sh -c "npm ci && npm run build"
```

## Troubleshooting

### `setup.sh: Permission denied`
```bash
chmod +x setup.sh
./setup.sh
```

### `The [public/storage] link already exists`
The setup script handles this automatically by removing the old symlink before creating a new one. If you run `storage:link` manually, remove the old one first:
```bash
docker compose exec app rm -rf public/storage
docker compose exec app php artisan storage:link
```

### Images not showing after setup
Check the volume is populated:
```bash
docker compose exec app ls storage/app/public/images/
```
If empty, run the image download manually:
```bash
docker compose exec -T app bash -c 'curl -L https://github.com/Ruuby666/bookingWeb/releases/download/images_v1/imagens.zip -o /var/www/html/imagens.zip && unzip /var/www/html/imagens.zip -d /var/www/html/storage/app/public/images/ && rm /var/www/html/imagens.zip'
```

### `Duplicate entry` error on seeder
The database volume still has data from a previous run. Do a full reset:
```bash
docker compose down -v
./setup.sh
```

### `no configuration file provided: not found`
You are not in the project root:
```bash
cd path/to/bookingWeb
```

### `Database file at path [...].sqlite does not exist`
Stale cached config. Fix with:
```bash
docker compose exec app php artisan optimize:clear
```
